TeamUp — Setup Guide

Files

File Where it goes in your project
teamup.qmd root of your Quarto project
teamup.css root of your Quarto project (same folder as teamup.qmd)
rebuild-teamup.yml .github/workflows/rebuild-teamup.yml

Step 1 — Create the Google Form

Create a Google Form with these fields (exact names don’t matter, but note column order):

Field Type Notes
Name Short answer
Skills Short answer Comma-separated, e.g. “Python, ML, SQL”
Experience level Multiple choice Beginner / Intermediate / Advanced
Role seeking Short answer e.g. “Looking for a data engineer”
Team status Multiple choice Looking for a team / Have a team, need 1 more / Team full
Contact (Discord/email) Short answer
Bio Paragraph 1–2 sentences

Step 2 — Link to a Google Sheet

In Google Forms → Responses tab → click the Sheets icon.
This creates a linked spreadsheet that updates in real time.

Step 3 — Make the Sheet public (read-only)

In Google Sheets → Share → “Anyone with the link” → Viewer.
This lets gs4_deauth() work without OAuth in the rendering environment.

Step 4 — Update teamup.qmd

Replace these two placeholders at the top of teamup.qmd:

SHEET_URL <- "YOUR_GOOGLE_SHEET_URL_HERE"
form_url  <- "YOUR_GOOGLE_FORM_URL_HERE"

Also update the column names in render_card() if your Google Sheet uses different header names (e.g. Timestamp gets added automatically by Forms — you can ignore it, it won’t be used).

Step 5 — Add the page to your _quarto.yml navigation

website:
  navbar:
    right:
      - text: "TeamUp"
        href: teamup.qmd

Step 6 — Enable GitHub Actions

Push the workflow file to .github/workflows/rebuild-teamup.yml.
The site will now rebuild every hour automatically.

To trigger a manual rebuild: GitHub repo → Actions tab → “Rebuild TeamUp page” → Run workflow.

Local preview

# In RStudio terminal:
quarto preview teamup.qmd
